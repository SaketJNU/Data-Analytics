Sources and Nature of Data:

1. Sources of Data

These are the origins from which data is collected. They are typically classified as:

a. Primary Data Sources

·         Definition: Data collected firsthand by the researcher for a specific purpose.

·         Examples:

o    Surveys and questionnaires

o    Interviews

o    Observations

o    Experiments

b. Secondary Data Sources

·         Definition: Data collected by someone else, used for a different purpose than originally intended.

·         Examples:

o    Government reports (e.g., census data)

o    Academic research papers

o    Company records

o    Online databases

o    News articles

2. Nature of Data

This refers to the type or form in which data exists. It can be categorized as:

a. Qualitative Data

·         Definition: Non-numerical data that describes qualities or characteristics.

·         Examples:

o    Opinions

o    Text responses

o    Interview transcripts

·         Types:

o    Nominal: Categories without any order (e.g., gender, religion)

o    Ordinal: Categories with a logical order (e.g., customer satisfaction levels)

b. Quantitative Data

·         Definition: Numerical data that can be measured and analyzed statistically.

·         Examples:

o    Age

o    Income

o    Test scores

·         Types:

o    Discrete: Countable values (e.g., number of children)

o    Continuous: Measurable values on a scale (e.g., height, temperature)

Introduction to Big Data Platform:

What is a Big Data Platform?

A Big Data Platform is an integrated software framework designed to handle, process, store, and analyze large volumes of data  structured, semi-structured, and unstructured  that traditional systems cannot manage efficiently.

It combines:

·         Data storage (like distributed file systems)

·         Processing engines (for batch or real-time analysis)

·         Analytics tools

·         Management and visualization layers

Evolution of Analytical Scalability

Analytical scalability refers to the ability of data analysis systems and processes to handle increasing amounts of data, users, or complexity efficiently. Over time, this capability has evolved significantly due to advances in technology, tools, and business needs.

Heres a breakdown of the evolution of analytical scalability:

1. Early Stage: Manual and Static Analysis (Pre-1990s)

  • Data Size: Kilobytes to megabytes.
  • Tools: Spreadsheets (e.g., Lotus 1-2-3, early Excel), paper-based reports.
  • Characteristics:
    • Limited to small datasets.
    • Static reports, updated infrequently.
    • Manual data entry and processing.
  • Scalability: Very limited.

 

2. Database Era: Relational Databases and SQL (1990s2000s)

  • Data Size: Megabytes to gigabytes.
  • Tools: SQL, MS Access, Oracle, MySQL.
  • Characteristics:
    • Structured data stored in relational databases.
    • Query-based access using SQL.
    • Batch processing for reports.
  • Scalability: Moderate  relational databases improved storage and access but struggled with very large or unstructured data.

3. Business Intelligence (BI) Era (2000s2010s)

  • Data Size: Gigabytes to terabytes.
  • Tools: BI tools like Tableau, Power BI, SAP BusinessObjects, Qlik.
  • Characteristics:
    • Interactive dashboards and visualization.
    • ETL (Extract, Transform, Load) pipelines for structured data.
    • Enterprise reporting and decision support.
  • Scalability: Improved  but mostly designed for structured data and periodic batch updates.

4. Big Data Era (2010sPresent)

  • Data Size: Terabytes to petabytes and beyond.
  • Tools: Hadoop, Spark, Hive, NoSQL (MongoDB, Cassandra), AWS, Azure, Google Cloud.
  • Characteristics:
    • Distributed computing frameworks (MapReduce, Spark).
    • Ability to store and process large volumes of structured and unstructured data.
    • Real-time and streaming analytics.
  • Scalability: High  distributed systems allow horizontal scaling across many servers.

5. AI & Cloud-Native Analytics Era (Present  Future)

  • Data Size: Petabytes to exabytes.
  • Tools: Snowflake, Databricks, BigQuery, ML platforms (TensorFlow, PyTorch), AutoML, GenAI tools.
  • Characteristics:
    • Serverless, on-demand cloud infrastructure.
    • Machine learning and AI integration for advanced analytics.
    • Auto-scaling, real-time processing, and intelligent insights.
  • Scalability: Very high  virtually unlimited, depending on cloud resources and design.

Analytic Process and Tools

The analytic process is a structured approach to extracting insights from data, and it typically involves several stages  from defining the problem to making data-driven decisions. Various tools are used at each stage to help manage, analyze, and visualize the data effectively.

Analytic Process: Key Steps

1. Define the Problem / Objective

·         Goal: Understand what question or business problem you're trying to solve.

·         Example: Why are customer churn rates increasing?

2. Data Collection

·         Goal: Gather relevant data from various sources.

·         Sources: Databases, APIs, spreadsheets, web scraping, IoT devices, CRM systems, etc.

3. Data Cleaning and Preparation

·         Goal: Remove errors, handle missing values, and format data properly for analysis.

·         Tasks: De-duplication, handling outliers, formatting columns, data transformation.

4. Data Exploration and Analysis

·         Goal: Discover patterns, trends, and relationships in the data.

·         Methods: Descriptive statistics, data visualization, correlation analysis.

5. Modeling and Advanced Analytics

·         Goal: Use statistical models or machine learning to predict or classify data.

·         Types:

o    Predictive (e.g., forecasting sales)

o    Prescriptive (e.g., optimizing pricing strategies)

o    Diagnostic (e.g., root cause analysis)

6. Data Visualization and Communication

·         Goal: Communicate insights effectively through visualizations or dashboards.

·         Outputs: Reports, dashboards, presentations.

7. Decision Making and Implementation

·         Goal: Use insights to inform strategies, automate processes, or guide actions.

8. Monitoring and Optimization

·         Goal: Track the results of decisions and refine models or processes as needed.

Data Analytics Life Cycle:

he Data Analytics Life Cycle is a structured approach to solving data-driven problems and delivering insights through analytics. It provides a roadmap for data professionals to manage and execute data projects efficiently.

Heres a breakdown of the 6 phases in the typical Data Analytics Life Cycle:

1. Discovery (Understanding the Problem)

Goal: Define the business problem and understand project goals.

Key Activities:

  • Identify business objectives.
  • Stakeholder interviews.
  • Assess available resources (people, data, technology).
  • Define the initial hypothesis or problem statement.

2. Data Preparation (Data Collection & Cleaning)

Goal: Collect, integrate, and clean data for analysis.

Key Activities:

  • Data collection (internal & external sources).
  • Data integration (combining data from multiple sources).
  • Data cleaning (handling missing values, duplicates, formatting).
  • Exploratory data analysis (EDA) to understand the data structure.

3. Model Planning

Goal: Decide the analytical techniques and tools to be used.

Key Activities:

  • Select appropriate models (e.g., regression, clustering, classification).
  • Determine data requirements for modeling.
  • Split data (e.g., training/test sets).
  • Choose tools (e.g., Python, R, SQL, Excel).

4. Model Building

Goal: Build and validate models.

Key Activities:

  • Build and train models.
  • Validate model performance (e.g., accuracy, precision, recall).
  • Tune hyperparameters.
  • Ensure model aligns with business goals.

5. Operationalize (Deploy the Solution)

Goal: Deliver final reports, dashboards, or deployed models.

Key Activities:

  • Prepare final deliverables (visualizations, reports).
  • Deploy models into a production environment (if applicable).
  • Present findings to stakeholders.
  • Provide documentation and user training.

6. Communicate Results / Feedback & Evaluation

Goal: Evaluate outcomes and gather feedback.

Key Activities:

  • Assess if business objectives were met.
  • Collect stakeholder feedback.
  • Monitor performance of deployed solution.
  • Iterate or improve the model if needed.
